# Control Maps

These maps provide traceability from SicarioSpec evidence to external control
and audit concerns. They are not certification claims and do not copy full
control text from third-party frameworks.

Current starter maps (14 frameworks):

- `ccm-v4.1-sicario.json`: CSA Cloud Controls Matrix v4.1 domain-level map.
- `sox-404-itgc-sicario.json`: SOX Section 404 / ICFR ITGC evidence map.
- `soc2-trust-services-sicario.json`: SOC 2 Trust Services Criteria evidence map.
- `fedramp-rev5-sicario.json`: FedRAMP Rev. 5 baseline and control-family evidence map.
- `bsi-c5-2026-sicario.json`: BSI C5:2026 cloud service criteria-area evidence map.
- `ssdf-800-218-sicario.json`: NIST SSDF (SP 800-218) practice-group evidence map.
- `ai-rmf-sicario.json`: NIST AI RMF (AI 100-1) Govern/Map/Measure/Manage evidence map.
- `iso-27001-2022-sicario.json`: ISO/IEC 27001:2022 Annex A theme + control-group map (4 themes, 93 controls).
- `nist-800-53-r5-sicario.json`: NIST SP 800-53 Rev 5 control-family map (20 families).
- `eu-ai-act-sicario.json`: EU AI Act risk-tier + high-risk obligation (Art. 9-15) map.
- `gdpr-cpra-sicario.json`: GDPR principles and duties map, with CPRA/CCPA consumer-rights parallels.
- `pci-dss-v4.0-sicario.json`: PCI DSS v4.0 12-requirement map.
- `hipaa-security-rule-sicario.json`: HIPAA Security Rule Administrative/Physical/Technical safeguard map.
- `owasp-asvs-sicario.json`: OWASP ASVS application security verification evidence map.

Each map is a starter that links SicarioSpec evidence to framework outcomes at a
coarse level (theme, domain, family, practice group, function, requirement, or
safeguard). They are not control-by-control crosswalks and not certification claims.

Use the official framework source when performing a real audit, attestation, or
customer-facing control assertion.
